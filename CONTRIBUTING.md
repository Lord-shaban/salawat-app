# دليل المساهمة في التطوير (Contributing Guidelines)

شكراً لاهتمامك بالمساهمة في تطبيق الصلوات المليارية! نحن نرحب بجميع المطورين والمصممين الذين يرغبون في تحسين هذا المشروع وكسب الأجر معنا.

## كيف يمكنك المساهمة؟

### 1. الإبلاغ عن الأخطاء (Bug Reports)
إذا واجهت أي مشكلة أو خطأ أثناء استخدام التطبيق، يُرجى فتح [Issue] جديد في المستودع واختيار القالب الخاص بـ (Bug Report). حاول توفير أكبر قدر ممكن من التفاصيل (خطوات الوصول للخطأ، نظام التشغيل، نوع المتصفح).

### 2. اقتراح مميزات جديدة (Feature Requests)
إذا كانت لديك فكرة رائعة لتحسين التطبيق، قم بفتح [Issue] جديد واختيار القالب (Feature Request). اشرح الفكرة وكيف ستفيد المستخدمين.

### 3. كتابة الكود وإصلاح الأخطاء (Pull Requests)

لكتابة الكود ورفعه، يرجى اتباع الخطوات التالية:

1. **عمل Fork** للمستودع (Repository).
2. **استنساخ المستودع** إلى جهازك الشخصي:
   ```bash
   git clone https://github.com/حسابك/salawat-app.git
   ```
3. **إنشاء فرع جديد (Branch)** للميزة التي تعمل عليها:
   ```bash
   git checkout -b feature/your-feature-name
   # أو
   git checkout -b fix/your-bug-fix
   ```
4. **كتابة كود نظيف ومفهوم**، وتأكد من الالتزام بكود النمط المُستخدم في المشروع. (تأكد من تشغيل أمر `npm run lint` للتأكد من عدم وجود أخطاء في الـ ESLint).
5. **إضافة التعديلات**:
   ```bash
   git add .
   git commit -m "feat: أضف ميزة جديدة كذا وكذا"
   ```
6. **رفع التعديلات للـ Fork الخاص بك**:
   ```bash
   git push origin feature/your-feature-name
   ```
7. **إنشاء طلب سحب (Pull Request)** من حسابك إلى المستودع الرئيسي. اكتب وصفاً جلياً للتعديلات التي قمت بها.

## قواعد مهمة
- حافظ على الأخلاق والتعامل الطيب مع جميع المساهمين (راجع [CODE_OF_CONDUCT.md](./CODE_OF_CONDUCT.md)).
- تأكد من إضافة كل المتغيرات البيئية الجديدة أو الناقصة في ملفات `README` إن وجدت لاحقاً.
- أسماء الفروع (Branches) يُفضل أن تكون واضحة ومسبوقة بكلمات مثل: `feat/`, `fix/`, أو `docs/`.

نسأل الله أن يجعل هذا العمل في ميزان حسناتك!
