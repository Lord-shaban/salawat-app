# Project Governance

This document outlines the governance model for **Salawat App** and describes how community members can participate, contribute, and take on leadership roles.

## 1. Principles

1. **Openness**: Anyone can participate, regardless of background.
2. **Transparency**: Project discussions, minutes, and decisions are visible to the community.
3. **Collaboration**: We value contributions of all forms: code, design, documentation, and answering questions.
4. **Resilience**: The architecture and community aim for extreme stability and scale.
5. **Purpose**: Every decision aligns with the goal of creating a dedicated, uninterrupted global counter.

## 2. Roles and Responsibilities

Our project uses a simplified, yet scalable governance model with standard roles:

### Contributors
Anyone who interacts with the project—reporting issues, creating pull requests, or participating in discussions.
- Create bugs and feature requests.
- Provide feedback on architectural decisions.
- Submit Pull Requests resolving documented issues.

### Maintainers
Active contributors who have shown dedication, technical acumen, and an understanding of our core mission.
- Triage issues and review/merge Pull Requests.
- Guide architectural priorities based on [ARCHITECTURE.md](ARCHITECTURE.md).
- Uphold the [Code of Conduct](CODE_OF_CONDUCT.md).

### Core Team
The founders and principal architects driving the vision.
- Set the overarching milestones and roadmap.
- Final say on high-risk technical decisions or large structural refactors.
- Have administrative access to Vercel/Upstash deployment environments.

## 3. Decision-Making Process
Decisions are driven by consensus. If consensus cannot be reached, the Core Team makes the final call. Major changes require written RFCs (Requests for Comments) in GitHub Issues to allow the community to digest and opine.

## 4. How to Become a Maintainer
There is no fixed timeline, but consistent, high-quality contributions over several months, paired with active community engagement and issue resolution, typically lead to maintainer status.
